# Stitch / 页面生成 Prompt 模板

复制下面模板，把 `[]` 替换成你的内容。建议用英文写结构说明，中文文案可直接保留中文。

```text
Design a [dark/light]-theme landing page for [产品名] — [一句话定位].

Target users:
[目标用户，用英文或中文描述]

Design vibe:
[调性，例如 Industrial / Minimal / Editorial / Japanese Minimal]
Keywords: [3-5 个形容词，例如 precise, trustworthy, developer-first, calm]

Typography:
Use [Display 字体] for headings.
Use [Body 字体] for body text.
Do not use Inter, Roboto, Arial, or system default fonts.

Color scheme:
Background: [背景色 HEX]
Surface: [卡片/区域色 HEX]
Primary accent: [主色 HEX]
CTA / highlight: [强调色 HEX]
Text primary: [文字色 HEX]
Text secondary: [次级文字 HEX]

Page structure:

1. NAVIGATION
Logo left, nav links center, primary CTA right.
Links: [Product], [Pricing], [FAQ], [Blog]
CTA: "[按钮文案]"

2. HERO SECTION
Layout: [split / asymmetric / embedded tool / dashboard preview]
Headline: "[定稿 Hero 标题]"
Subheadline: "[定稿 Hero 副标题]"
Primary CTA: "[主 CTA]"
Secondary CTA: "[次 CTA]"
Hero visual: [产品截图 / 输入框 demo / dashboard / before-after]

3. PROBLEM / PAIN
[痛点区标题]
[痛点描述]
Use [布局：cards / split / timeline]

4. SOLUTION / BENEFITS
Show [N] benefit cards:
- [Benefit 1]: [描述]
- [Benefit 2]: [描述]
- [Benefit 3]: [描述]

5. HOW IT WORKS
3 steps:
1. [步骤 1]
2. [步骤 2]
3. [步骤 3]

6. FEATURES
Use [bento grid / 2+1 / asymmetric cards].
Features:
- [Feature 1]
- [Feature 2]
- [Feature 3]
- [Feature 4]

7. PRICING
Plans:
- [Plan 1] — [价格] — [权益]
- [Plan 2] — [价格] — [权益]
Highlight [推荐 Plan].

8. FAQ
Questions:
- [FAQ 1]
- [FAQ 2]
- [FAQ 3]

9. FINAL CTA
Repeat the core promise with a stronger action.
CTA: "[按钮文案]"

10. FOOTER
Logo, Privacy, Terms, Contact, copyright.

Design constraints:
- NOT a generic SaaS template.
- No purple-blue gradient on white background.
- No centered hero with generic 3-column features.
- No emoji icons; use Material Icons, Lucide, or custom SVG icons.
- Use asymmetric layout where appropriate.
- CTA must be the most visible element above the fold.
- Keep strong information hierarchy.
- Make the design look like a real brand, not AI-generated.

Device:
This is the [DESKTOP/MOBILE] version.
For mobile: single column, headline max 2 lines, cards stack vertically, minimum touch target 44x44px, no horizontal overflow.
```
